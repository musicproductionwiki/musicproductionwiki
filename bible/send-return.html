Send & Return

/sɛnd ænd rɪˈtɜːrn/

Quick Answer Definition How It Works Parameters Quick Reference History How To Use It In The Wild Types Mistakes Related Further Reading FAQ

Send & Return is a parallel signal routing method where a copy of a channel's signal is tapped off, processed through a shared effect, and blended back into the mix. It preserves the dry signal while allowing multiple tracks to share one effect instance.

01 Definition

Every legendary room sound you've ever heard on a record — Bono's vocal floating in a cathedral, John Bonham's snare cracking through Headley Grange — was built on one quietly powerful idea: send a little signal somewhere else, and let it come back changed.

Send & Return is a parallel signal routing architecture used throughout mixing and recording. Rather than placing an effect directly in a channel's insert slot — where it processes 100% of the signal in series — the Send & Return method taps a copy of the dry signal from one or more source channels, routes that copy to a dedicated auxiliary (aux) bus or return track, processes it through an effect set to 100% wet output, then blends the processed signal back alongside the untouched originals. The result is additive: the dry signal remains completely intact while the effected signal layers on top, giving the engineer precise and continuous control over depth and blend without ever altering the source itself.

The architecture has two named poles. The send is the variable-level tap point, typically a knob or fader on each source channel that controls how much of that channel's signal is fed to the shared bus. Multiple sends from different tracks can feed the same bus simultaneously, meaning a single reverb unit can service an entire drum kit, a vocal stack, or an entire session, with each contributor sending independently controllable amounts. The return — also called an aux return, FX return, or return track depending on the DAW — is the bus channel that receives all incoming send signals, hosts the effect plugin or hardware unit, and contributes its processed output to the main mix at a gain level set by the return fader.

The practical implications of this architecture are profound. Because the effect lives on one return channel rather than duplicated across many inserts, CPU load is reduced, recall is instantaneous (change the return plugin once, the change applies to every sender), and the tonal relationship between all sending tracks stays coherent — they all share the same acoustic space. This coherence is why engineers reach for Send & Return for time- and space-based effects above all else: reverb, delay, chorus, and room simulation are inherently about placing sounds in a shared environment, and a shared bus enforces that shared-space logic at the routing level.

Send & Return also underpins several advanced processing techniques that have nothing to do with reverb. Parallel compression — often called New York compression — uses a send to feed a heavily compressed copy of a drum bus or vocal back alongside the dry signal, adding density without killing transients. Parallel saturation, parallel EQ, and even parallel pitch-shifting follow the same logic. Sidechain routing frequently travels through send paths. In larger sessions, stem buses and submix architectures borrow Send & Return principles to create hierarchical mixing structures. Understanding the Send & Return at a conceptual level is therefore not merely about knowing where to put your reverb: it is foundational literacy for every advanced mixing workflow.

02 How It Works

At the circuit level, a send is a bus assignment with a gain control preceding it. On an analog console, this is implemented as a potentiometer on each channel strip that taps the channel signal post-fader (or pre-fader, depending on the switch position) and routes it to a summing bus. Every channel contributing to that bus has its own independent send pot, and the summed result feeds the aux output jack. The aux output connects — physically via a patch cable — to the input of an outboard effect unit. The effect's output returns to a dedicated return channel on the console, which has its own fader, pan pot, and routing assignments. The return fader controls how loudly the processed signal sits in the main stereo bus.

Two send modes define fundamentally different behaviors. A post-fader send taps the signal after the channel fader has been applied, meaning the send level tracks the channel level. As you ride the vocal fader down, the reverb send automatically decreases proportionally — the tail of the effect breathes with the performance. This is the default for reverb and delay in virtually all professional mixing workflows because it maintains a consistent wet-to-dry ratio regardless of dynamic rides. A pre-fader send taps before the channel fader, meaning the send level is independent of the channel fader. Pulling the channel fader to minus infinity does not silence the effect send; the reverb tail or delay continues to arrive at the return bus. Pre-fader sends are used for headphone cue mixes, for Haas-effect widening tricks where the dry signal is muted on the main mix, and for creative effect-without-source textures.

In DAWs, the architecture is software-emulated but functionally identical. In Ableton Live, sends are the A/B/C/D knobs on each track, and Returns are the dedicated return tracks in the master section. In Pro Tools, auxiliary input tracks receive signal via bus assignments made in each source channel's output or send assignment. Logic Pro uses Sends routed to Bus channels that appear as Aux tracks. FL Studio's Mixer uses the send routing matrix at the bottom of each mixer channel. Reaper implements sends via the track routing matrix. Despite different UI paradigms, the underlying signal graph is always the same: one signal copied, one effect shared, one return blended.

Plugin configuration on a return track requires one non-negotiable setting: the effect's internal mix or dry/wet control must be set to 100% wet (zero dry signal). Because the return track is receiving only the send copy — not the original dry signal — any dry signal passing through the plugin would double the source channel's dry audio at the mix bus, causing comb filtering or unintended phase issues. The blend between dry and wet is managed entirely by the send and return fader levels, not by the plugin's internal mix knob. This is the single most common misconfiguration beginners make, and understanding it separates competent routing from the muddy, undefined mixes that result from treating return tracks like insert chains.

03 The Parameters

Every send & return — hardware or plugin — operates on the same core parameters. Know these and you can work with any implementation.

SEND LEVEL
Controls how much signal is tapped and routed to the return bus

Typically calibrated in dB, ranging from minus infinity (off) to 0 dB or above on some consoles. At −10 dB a moderate amount of dry signal feeds the reverb; at 0 dB the full pre-fader or post-fader level is sent. Subtle verb tails on vocals typically live between −20 dB and −12 dB; dense washes on pads might push to −6 dB. This is the primary blend control and should be adjusted while listening to the full mix in context.

PRE/POST FADER
Determines whether the send taps before or after the channel fader

Post-fader is standard for reverb and delay in music mixing — the effect depth follows fader rides, preserving wet/dry ratio. Pre-fader is used for headphone cue mixes, creative effects-without-source techniques, and certain parallel processing scenarios where the effect must remain constant regardless of the dry level. Choosing incorrectly produces reverb tails that persist after a fade-out (pre-fader) or disappear too quickly during dynamic automation (post-fader used where pre was needed).

RETURN FADER
Sets the overall contribution of the processed signal in the main mix

After all sends are summed and processed through the effect, the return fader controls the final blend level before the signal reaches the main bus. Unlike individual send knobs, which adjust each source's contribution independently, the return fader is a master wet level for the entire bus. Pulling the return down at the end of a song is a clean way to automate a reverb fade without touching individual track automation. Unity gain (0 dB) on a return track at mix time is common for prominent effects; −6 to −12 dB is typical for ambient verb tails.

EFFECT WET/DRY MIX
Plugin-internal blend — must be 100% wet on return tracks

When a plugin is placed on a return track, its internal dry/wet mix parameter must be set to 100% wet. Any dry signal passed through the plugin adds a time-aligned copy of the source to the mix bus, causing phase cancellation or low-frequency comb filtering. The exceptions are creative insert uses of reverb (when you deliberately want a 100% wet channel contributing to a wet layer) or M/S processors where mid and side channels are treated separately. Always verify this setting when setting up a new return track.

SEND BUS CHANNEL SELECTION
Assigns which return bus the send feeds

In DAWs, each send must be assigned to a specific bus or return track. A session may have a room reverb return, a hall reverb return, a slapback delay return, a plate reverb return, and a parallel compression return all operating simultaneously. Each source track can send to multiple buses independently, with individual level control per bus. Keeping bus assignments organized — labeled clearly and color-coded — is a professional workflow essential that pays dividends during automation, recalls, and mix revisions.

SEND PAN
Positions the send signal in the stereo field before it reaches the bus

Some consoles and DAWs offer a pan control on the send itself, independent of the channel pan. This allows a mono source to be panned differently within the effect's input — for example, centering the reverb send of a hard-panned guitar so that the reverb bloom appears in the center of the mix while the dry guitar sits at the side. In most DAW workflows the send inherits the channel pan by default, but checking and adjusting send pan is a powerful spatial design tool.

05 History & Origin

The send and return concept predates the multitrack era entirely. By the early 1950s, recording studios at Capitol, RCA Victor, and Columbia had already begun routing signals from their recording consoles to physical echo chambers — tiled rooms equipped with a loudspeaker and a distant microphone — via separate send outputs. The chamber microphone's signal returned to the console on a dedicated input, and an operator adjusted a rotary gain control to blend the reverberated signal with the dry recording. This was mechanically identical to the modern software implementation: a tapped copy, a parallel processing environment, a blended return. The only difference was that the processing unit was a reverberant room rather than a plugin.

The formalization of the auxiliary send as a console feature occurred during the 1960s as British and American console manufacturers recognized that recording facilities needed systematic, repeatable ways to route multiple sources to shared outboard effects. Neve's 8014 console (circa 1970) and its descendants incorporated dedicated aux send busses with per-channel knobs, while Rupert Neve's designs for Neve Electronics set the electrical standard that most subsequent console makers emulated. The SSL 4000 series, introduced in 1976 and rapidly adopted by commercial studios throughout the late 1970s and 1980s, standardized the pre/post fader send switch, the channel-assign buttons, and the return-level fader architecture that DAW developers would later replicate in software. Engineers like Bruce Swedien (Michael Jackson's Thriller), Bob Clearmountain (Bruce Springsteen's Born in the USA), and Hugh Padgham (Phil Collins, Peter Gabriel) developed distinct aux routing philosophies on SSL and Neve consoles that defined the sound of commercially dominant pop records for two decades.

Parallel compression — arguably the most important non-reverb application of Send & Return — was popularized in New York in the 1980s and 1990s and became known as the New York compression technique. Engineers at studios like Electric Lady, Right Track, and Unique Recording discovered that sending a drum bus copy to a heavily compressed return and blending it below the dry signal added density, sustain, and punch without the pumping artifacts of direct insert compression. Bob Power, mixing for A Tribe Called Quest and D'Angelo, and Andy Wallace, mixing Nirvana and Rage Against the Machine, both documented versions of this technique in interviews throughout the 1990s. The technique became a staple of every genre from hip-hop to rock within a decade.

When digital audio workstations matured in the early 2000s, the send and return architecture migrated from hardware consoles into software environments essentially unchanged in concept but dramatically expanded in scope. Digidesign's Pro Tools TDM system (which became Pro Tools HD) allowed engineers to route signals to Auxiliary Input tracks via internal bus assignments, effectively replicating the SSL console routing in software. Steinberg Cubase, MOTU Digital Performer, and Logic Audio (before its Apple acquisition) all developed equivalent systems simultaneously. The democratization of DAW production from roughly 2005 onward — driven by affordable interfaces, powerful consumer CPUs, and DAWs priced accessibly — put the full Send & Return workflow into the hands of bedroom producers worldwide, removing the last barrier that had historically confined this fundamental technique to commercial studio environments.

06 How Producers Use It

Drums and percussion are the most common Send & Return application in contemporary production. A typical drum session routes multiple sources — kick, snare, hi-hats, toms, overheads, and room mics — to two or three shared reverb returns: a short room (30–80 ms pre-delay, 0.4–0.8 s RT60) for glue and dimensional presence, a medium hall (80–150 ms pre-delay, 1.2–2.0 s RT60) for snare bloom, and sometimes a large plate for hi-end shimmer. Each drum element sends a precisely calibrated amount — kick typically near silence or absent, snare heavily sent, overheads moderately sent — creating a unified acoustic environment without smearing transient attack. A parallel compression return additionally receives the drum bus copy at 0 dB send, processed with 40–60 dB of gain reduction using a Urei 1176 or its software equivalent, blended back at −10 to −6 dB return level.

Vocals represent the most dynamically critical Send & Return application. A lead vocal typically feeds two returns: a short pre-delay reverb (20–40 ms pre-delay, 0.8–1.5 s RT60) for perceived size without distance, and a quarter-note tempo-synced delay for rhythmic depth. Because the vocal is the most exposed element in any mix, the relationship between send level and return fader is often automated — the reverb send may be increased on long held notes and reduced during dense verse sections. Engineers like Chris Lord-Alge and Tom Lord-Alge, both prolific users of the SSL console aux architecture, built their vocal sounds on exactly this dynamic send automation approach, which creates the illusion that the reverb responds musically to the performance.

Parallel compression buses apply Send & Return principles to dynamic control rather than spatial processing. The technique is equally valid in software and hardware: the source signal (typically a drum bus, vocal bus, or master bus) is sent in full at 0 dB pre-fader to a return track hosting a compressor set to extreme ratios (8:1 to 20:1) and fast attack/release times. The resulting crushed signal is blended back at −12 to −6 dB, adding body and sustain invisibly behind the dynamic transients of the dry signal. In electronic music, this technique applied to a full program bus — sometimes called the New York master — is a low-cost alternative to dedicated stem compression and is widely used in techno, drum and bass, and hip-hop production.

Creative and genre-specific uses extend well beyond conventional reverb and delay. In dub and reggae production, send levels are actively performed during mixdown — manually riding send knobs in real time to create dub echo effects where delay feeds swell dramatically and then retreat. In ambient and experimental music, pre-fader sends are used deliberately: the dry source is faded out of the mix while the reverb tail continues to bloom, creating the ghostly decayed-source effect central to the genre. In modern pop and R&B production, multiple pitch-shifted return tracks fed from a single vocal send create complex harmonically-rich vocal cloud effects that would be impractical to implement with individual insert chains on every duplicate track.

07 In the Wild

Abstract knowledge becomes practical when you can hear it in music you know. These tracks demonstrate send & return used intentionally, at specific moments, for specific purposes.

Led Zeppelin — "When the Levee Breaks" (1971)
0:00 onward · Produced by Jimmy Page

John Bonham's drum kit was recorded in the stairwell of Headley Grange using microphones placed on the upper landing, with the room's natural reverberation captured directly. However, during mixdown, additional aux returns were used to add plate reverb to specific elements. The distant, cavernous snare — arguably the most sampled drum sound in recorded history — demonstrates the principle of shared spatial environments: the room return unifies the entire drum performance in a single acoustic space. Listen for how the reverb tail on the snare extends through the decay without obscuring the transient attack of the subsequent hit.

Michael Jackson — "Thriller" (1982)
0:18 vocal entrance · Produced by Quincy Jones, engineered by Bruce Swedien

Bruce Swedien's vocal chain on Thriller is documented as involving multiple aux sends: a close, intimate ambience return (short pre-delay plate) and a larger hall return used selectively on sustained phrases. At 0:18 when Jackson's lead vocal enters, the short plate reverb is clearly audible as a tight bloom around each syllable — present enough to add dimension but short enough to preserve lyric intelligibility. During the chorus, the hall send appears to increase, giving the multi-tracked backing vocals a dramatically larger environment. This dynamic send approach — changing reverb depth based on the song's structural moment — became a template for 1980s pop vocal mixing.

A Tribe Called Quest — "Electric Relaxation" (1993)
Full track · Produced by Q-Tip, mixed by Bob Power

Bob Power's mix demonstrates parallel compression Send & Return on the drum loop. The original loop — sampled from Ronnie Foster's "Mystic Brew" — retains its crisp transients while sitting in a cushion of compressed density that is characteristic of the New York parallel compression technique. This warmth is not achievable with direct insert compression at equivalent ratios, which would suppress the snare transient. Power's send routing approach gives the drums simultaneous punch and body. Listen through headphones to hear how the sustain of each drum hit extends beneath the dry snap.

Radiohead — "How to Disappear Completely" (2000)
Throughout · Produced by Nigel Godrich

Nigel Godrich's mixing on Kid A exemplifies the pre-fader send technique used creatively. On several string and vocal sections, the source signal fades from the main mix while reverb and delay return signals continue to bloom, creating the haunted, sourceless quality that defines the album's emotional texture. At approximately 1:45, Thom Yorke's vocal appears to dissolve into its own reverb rather than fade cleanly — a direct result of return signals outlasting the dry signal, achievable only with pre-fader send routing or independent return automation.

Kendrick Lamar — "HUMBLE." (2017)
0:00 kick onset · Produced by Mike WiLL Made-It

The opening kick drum on HUMBLE. illustrates Send & Return parallel compression applied to a single element. The sub-frequency punch is massive and sustained far beyond what direct insert compression would allow, while the transient click remains sharp and well-defined — the classic parallel compression signature. Mike WiLL Made-It's drum bus routing reportedly involved multiple parallel layers at different compression depths blended by return fader level, creating a drum body that translates on both club systems and earbuds simultaneously.

08 Types & Variants

Reverb Return Bus
Lexicon 480L · EMT 140 Plate · AMS RMX16

The most ubiquitous Send & Return application, routing one or more tracks to a shared reverb unit or plugin set to 100% wet. Multiple reverb return buses in a session allow sources to inhabit different acoustic environments simultaneously — a drum room return, a vocal hall return, and a FX plate return can coexist. Each source selects which environments it contributes to by choosing which send buses to activate.

Delay Return Bus
Roland RE-201 Space Echo · Eventide H3000 · TC Electronic 2290

A dedicated delay return bus allows tempo-synced or free-running delays to be shared across multiple instruments without taxing CPU with duplicate plugin instances. Quarter-note and dotted-eighth delays on separate return buses — both fed from the same vocal send at different levels — create the rhythmic depth layering central to modern pop and R&B production. Pre-delay on the reverb return is often routed through the delay return first for compound spatial effects.

Parallel Compression Bus
Urei 1176LN · dbx 160 · SSL G-Bus Compressor

A return track receiving a 0 dB send from a drum bus, vocal bus, or full program, processed through a compressor at extreme settings (high ratio, fast attack, slow release), then blended back at reduced level. The technique adds density and sustain invisible to the ear as compression, because the unaffected dry signal dominates transient character. This is the New York compression technique and is standard practice on professional drum and full-mix buses.

Parallel Saturation and Distortion Bus
Neve 1073 preamp saturation · Ampex ATR-102 tape · Thermionic Culture Vulture

A return bus hosting a saturation plugin (Soundtoys Decapitator, Softube Saturation Knob, or tape emulation) receives sends from clean digital sources — synthesizers, drum machines, programmed percussion — to add harmonic warmth without committing the source to permanent saturation. This is especially useful in electronic music where clean oscillator sources benefit from upper harmonic content at specific blend levels that can be adjusted post-session.

Cue Mix / Headphone Bus
Console aux outputs · Furman HDS-16 · Aviom AN-16/i

A pre-fader send application where performers in a tracking session receive a custom mix through headphones independent of the main mix fader positions. Because the sends are pre-fader, an engineer can mute a track from the main mix without eliminating it from the performer's headphone feed. Each performer can have their own unique aux cue mix, with independent send levels adjusted specifically for their monitoring preference.

09 Common Mistakes

  • ✕
    Leaving plugin wet/dry mix below 100% on a return track
    The most common beginner error: placing a reverb on a return track but leaving the internal mix at 50% (the typical plugin default). This passes 50% dry signal through the return, doubling the source at the mix bus and causing comb filtering and low-frequency buildup that is almost impossible to diagnose without examining individual track outputs. The fix is immediate and absolute: set every plugin on every return track to 100% wet before touching any other parameter. Make this a pre-session checklist item.
  • ✕
    Using insert reverb instead of return for elements needing shared space
    Placing individual reverb inserts on every drum track instead of using a shared return bus creates multiple slightly different reverb characters from the same algorithm — particularly if plugin instances are initialized with different random seed values or have accumulated minor CPU-timing divergences. The result is a drum kit that sounds like it was recorded in several different rooms simultaneously. Shared return buses enforce a single acoustic environment across all contributing elements, which is essential for drum kit, string section, and backing vocal coherence.
  • ✕
    Setting sends post-fader then automating faders for effects automation
    Because post-fader sends track channel fader positions, automating a channel fader to zero (for a fade or mute) simultaneously silences the reverb send. If the intent was to automate only the dry level while keeping the reverb tail active, post-fader sends will prevent that behavior. For fade-outs that require the reverb or delay tail to ring out after the dry signal disappears, switch to pre-fader sends, automate the channel fader to minus infinity, and let the return tail decay naturally.
  • ✕
    Overloading a single return bus with sends from too many tracks
    Sending the full session — 30+ tracks — to a single reverb return at moderate-to-high levels produces a dense smear of reverberated signal that accumulates several dB above the nominal mix level and masks transients across the entire frequency spectrum. Professional practice uses 2–4 return buses differentiated by size and character, with only contextually appropriate tracks contributing to each. A room return might receive drums, percussion, and bass guitar; a hall return serves vocals and keys; a plate return handles individual effect moments.
  • ✕
    Neglecting return track EQ and high-pass filtering
    Reverb return buses accumulate low-frequency energy from every contributing track, and this energy has nowhere to go except into the mix bus. A hall reverb return receiving sends from kick drum, bass guitar, and low-end synthesizers will add muddy sub and low-mid buildup that can raise the perceived noise floor of the entire mix. High-pass filtering every return bus at 80–200 Hz (depending on the highest-fundamental source contributing) and adding a high-shelf rollout above 10–12 kHz keeps reverb air while preventing mix-muddying accumulation.
  • ✕
    Ignoring send level automation throughout the mix
    Static send levels set at the start of a mix and never revisited produce reverb depth that is equally present in quiet verses and dense choruses, which is acoustically implausible and makes the mix sound flat. Professional vocal send automation, for example, typically increases reverb send level on held notes and final syllables of phrases, and reduces it during dense syllabic passages where reverb would reduce intelligibility. Send level automation is as important as fader automation for creating a mix that breathes and has dynamic emotional range.

A drum bus, for example, can simultaneously feed the main mix and a reverb return bus. The reverb return bus can in turn send to a parallel compression return. This creates layered processing hierarchies that would be impossible with insert-only chains. In Reaper and Pro Tools in particular, multi-stage return routing is especially flexible, though care must be taken to avoid feedback loops — a return accidentally sending back into its own source chain.
In virtually all DAWs, send knobs and faders are automatable parameters identical to channel faders. In Ableton, right-click any Send knob and select "Show Automation" to write send level automation in the Arrangement view. In Pro Tools, sends have their own automation lanes accessible from the Track View selector. In Logic, send parameters appear in the Track Automation menu. Automate reverb sends to increase on long sustained notes and phrase endings, and decrease during dense articulated passages, to create a mix that breathes dynamically rather than sitting in static reverb depth throughout.
