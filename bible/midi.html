MIDI

/ˈmɪdi/

Quick Answer Definition How It Works Parameters Quick Reference History How To Use It In The Wild Types Mistakes Related Further Reading FAQ

MIDI is a digital communication protocol that transmits musical performance data — note pitch, velocity, timing, and control information — between instruments, computers, and hardware, without carrying any audio signal.

01 Definition

Every note you've ever programmed, every synth you've ever triggered remotely, every automation curve you've ever drawn — all of it runs on a 40-year-old protocol that fit on a single page of spec sheet. Understanding MIDI at the bit level is the difference between fighting your tools and making them do exactly what you imagine.

MIDI — Musical Instrument Digital Interface — is a serial communication protocol and file format standard first ratified in 1983 that allows electronic instruments, computers, controllers, and stage equipment to exchange performance instructions in real time. It does not transmit sound; it transmits intent. A MIDI message says "play the note C4 at velocity 95 for 480 milliseconds" and leaves the sound generation entirely to the receiving device. This separation of performance data from audio is MIDI's foundational genius and the reason the same protocol that drove a Roland D-50 in 1987 still routes flawlessly through a 2025 software instrument on an M-series Mac.

Structurally, the MIDI 1.0 specification defines messages as sequences of 8-bit bytes transmitted at 31,250 baud over a unidirectional current loop. Each message begins with a status byte — its most significant bit set to 1 — followed by one or two data bytes whose MSB is always 0. This architecture allows a receiver to re-sync to the data stream at any message boundary, making the protocol remarkably robust to cable quality and timing jitter. The original five-pin DIN connector carried this serial stream with opto-isolation to prevent ground loops between stage equipment — a practical engineering decision that has kept the hardware standard alive across four decades of consumer electronics.

MIDI messages fall into two principal categories: Channel Messages and System Messages. Channel Messages are further divided into Voice Messages — which carry performance data like Note On, Note Off, Pitch Bend, and Channel Pressure — and Channel Mode Messages, which configure how a device responds to incoming data (monophonic vs. polyphonic, Omni mode, etc.). System Messages include System Common messages for song position and time code, System Real-Time messages for clock and transport control (MIDI Clock, Start, Stop, Continue), and System Exclusive (SysEx) messages that allow manufacturers to transmit arbitrary device-specific data within the MIDI stream. Together these message types give MIDI enough expressivity to control an entire studio from a single sequencer output.

MIDI 2.0, ratified by the MIDI Association in 2020 and adopted by major OS vendors and DAW developers by 2023, dramatically extends the protocol's resolution and bidirectionality. Where MIDI 1.0 offers 7 bits (128 steps) of velocity and controller resolution, MIDI 2.0 expands these to 16-bit and 32-bit fields, providing over 65,000 and 4 billion discrete steps respectively. MIDI 2.0 also introduces the MIDI Capability Inquiry (MIDI-CI) handshake, allowing devices to negotiate features and share profile and property data bidirectionally — something MIDI 1.0 could never do. For producers, this means future hardware and plugins will express nuance in velocity, aftertouch, and continuous controllers that was previously impossible to encode.

02 How It Works

At the electrical level, MIDI 1.0 uses a 5 mA current loop at 31,250 bits per second with one start bit, eight data bits, and one stop bit per byte — a format called 8-N-1 asynchronous serial. The receiving device's input is opto-isolated, meaning the current physically drives an LED that triggers a phototransistor, breaking the ground connection between sender and receiver. This eliminates hum-inducing ground loops between rack-mounted gear. The 31,250 baud rate was chosen because it is exactly half of 1 MHz divided by 16, making it trivially derivable from the CPU clock speeds common in early-1980s microcontrollers. Over USB, MIDI is encapsulated in 4-byte USB packets using the USB MIDI Class 1 specification, removing the speed limit and allowing multiple virtual cables within a single physical connection.

A Note On message consists of three bytes: a status byte encoding both the message type (Note On = 0x9) and the MIDI channel (0–15, displayed to users as 1–16), a data byte for the note number (0–127, where middle C is 60), and a data byte for velocity (1–127; a velocity of 0 on a Note On is interpreted as Note Off). The 16-channel limit in MIDI 1.0 means a single MIDI cable can address up to 16 independent instruments simultaneously — enough for full General MIDI orchestral setups but a constraint for large modular or live-looping rigs, which often chain multiple MIDI interfaces. Pitch Bend messages use two data bytes combined into a 14-bit value (0–16383, center at 8192) giving 8,192 steps of resolution in each direction — far smoother than the 128-step controllers but still audible as stepping at slow rates on highly tuned synths.

MIDI Clock (System Real-Time message 0xF8) is transmitted 24 times per quarter note, regardless of tempo. A receiving device counts these pulses and derives tempo from the interval between them. MIDI Time Code (MTC) translates SMPTE timecode into MIDI messages, allowing frame-accurate sync between a DAW and external video or tape equipment. Song Position Pointer (SPP) tells a device which 16th-note beat position the master is currently at, enabling a slaved device to jump to mid-song positions rather than always starting from the top. These sync messages are foundational to live performance rigs and multi-room studio setups where hardware sequencers, drum machines, and DAWs must stay locked together.

Continuous Controller (CC) messages carry two data bytes: a controller number (0–127) and a value (0–127). The MIDI spec reserves CC numbers 0–31 for MSB (Most Significant Byte) controllers and 32–63 for their corresponding LSBs, enabling 14-bit resolution (16,384 steps) on critical parameters like modulation and volume when both bytes are transmitted. In practice, most devices send only the MSB, giving 128 steps. Controllers 64–69 are defined as on/off switches (sustain pedal is CC 64). Controllers 120–127 are Channel Mode Messages (All Notes Off, All Sound Off, etc.). Numbers 70–119 are undefined in the original spec and have been assigned by manufacturers for parameters like filter cutoff, resonance, and envelope attack — creating the de-facto standards that most DAWs and plugins recognize by name.

In a modern DAW session, MIDI data lives in a piano-roll editor as a sequence of note objects and automation lanes. When the transport plays, the DAW's MIDI engine reads these events and dispatches them to virtual instruments or hardware ports with sample-accurate timing — meaning the MIDI event is scheduled to fire at a specific audio buffer sample boundary, not merely "soon." This sample-accurate scheduling, combined with plug-in delay compensation (PDC), eliminates the timing drift that plagued hardware MIDI chains of the 1990s. The MIDI track itself stores no audio; the instrument or external device receiving the MIDI renders audio independently, which is why a single MIDI performance can be instantly redirected to a completely different sound by changing the destination instrument.

03 The Parameters

Every midi — hardware or plugin — operates on the same core parameters. Know these and you can work with any implementation.

NOTE NUMBER
Pitch identity encoded as an integer from 0 to 127

Middle C (C4) is note 60 in most DAWs, though some use C3 depending on octave convention. Each integer step represents one semitone; the full range spans just over ten octaves (C-1 to G9). In MIDI 2.0, note number resolution remains 8-bit but per-note pitch bend provides continuous microtonal control within each semitone.

VELOCITY
Force or speed of a key or pad strike, controlling amplitude and timbre

Encoded as a 7-bit value from 1 to 127 — a velocity of 0 on a Note On message is interpreted as Note Off. Most samplers and synths map velocity to amplitude and often to filter cutoff, sample layer switching, or envelope depth. Humanising MIDI velocity — randomising it ±5–15 around a target — is the single fastest way to remove the machine-gun effect from quantised patterns.

CHANNEL (1–16)
Logical routing address within a single MIDI cable or port

Each MIDI port carries 16 independent channels, allowing 16 separate instruments to coexist on one physical connection. Channel 10 is universally reserved for General MIDI percussion, which is why drum machines default to it. In a DAW, channel assignment routes MIDI to a specific instrument track or hardware port channel without requiring additional cables.

CONTINUOUS CONTROLLER (CC)
Real-time parameter automation using numbered controller slots

CC messages use a controller number (0–127) and a value (0–127), giving 128 steps of resolution for parameters like modulation (CC 1), volume (CC 7), pan (CC 10), expression (CC 11), and sustain pedal (CC 64). Combining MSB (0–31) and LSB (32–63) pairs yields 14-bit resolution (16,384 steps) for smoother automation on critical parameters. In a modern DAW automation lane, CC data is typically drawn or recorded rather than transmitted in real time, but it remains the backbone of plugin macro control.

PITCH BEND
Fine-grained pitch deviation encoded as a 14-bit bipolar value

Pitch Bend is a dedicated message type (not a CC) using two data bytes combined into a value from 0 to 16,383, with 8,192 as centre. The actual pitch deviation in semitones is determined by the receiving instrument's Pitch Bend Range setting (typically ±2 semitones but often ±12 or 48 on synths). At slow rates, the 128-step quantisation of each byte can produce audible stepping — MIDI 2.0 replaces this with a 32-bit field, eliminating the artefact entirely.

AFTERTOUCH (CHANNEL / POLY)
Pressure-sensitive modulation applied after a note is struck

Channel Aftertouch sends a single pressure value for all active notes, while Polyphonic Aftertouch (Poly AT) sends individual pressure values per note — a feature supported by instruments like the Ensoniq ESQ-1 and modern MPE controllers like the ROLI Seaboard. Aftertouch is commonly mapped to vibrato depth, filter cutoff, or amplitude. Many cheap keyboards lack aftertouch altogether; producers working in expressive genres should verify controller support before programming AT-heavy parts.

MIDI CLOCK & TRANSPORT
Timing synchronisation messages that lock multiple devices to one tempo

MIDI Clock (0xF8) fires 24 times per quarter note regardless of tempo, allowing slaves to derive BPM from pulse interval. Start (0xFA), Stop (0xFC), and Continue (0xFB) control transport state across devices. Song Position Pointer (0xF2) transmits the current 16th-note position so a slave can jump to mid-song locations. Tight MIDI clock sync requires a low-latency host; USB hubs and multi-hop MIDI chains introduce jitter that degrades groove at tempos above ~140 BPM.

SYSTEM EXCLUSIVE (SYSEX)
Manufacturer-defined data packets for deep device configuration

SysEx messages begin with 0xF0 and end with 0xF7, encapsulating an arbitrary payload identified by a manufacturer ID. They carry patch dumps, firmware updates, parameter edits, and tuning tables that go beyond the fixed CC structure. In a DAW, SysEx can be inserted directly into a MIDI track to automate synth parameters that have no CC mapping. Parsing SysEx requires manufacturer documentation or a MIDI SysEx library, as there is no universal structure beyond the framing bytes.

05 History & Origin

The MIDI specification emerged from a crisis of incompatibility. By the early 1980s, synthesizer manufacturers — Roland, Moog, Sequential Circuits, Oberheim, Korg, Yamaha — had each developed proprietary digital control interfaces that could not communicate with one another. Dave Smith of Sequential Circuits proposed a universal standard at the October 1981 Audio Engineering Society convention in New York, presenting a paper titled "A Proposed Universal Synthesizer Interface." The proposal attracted industry-wide interest, and Smith collaborated with Ikutaro Kakehashi, founder of Roland Corporation, to refine the electrical and protocol specifications. The first public MIDI demonstration took place at the January 1983 NAMM Winter Show, where a Sequential Circuits Prophet-600 and a Roland JP-6 were connected and played in unison — the handshake worked on the first attempt, a moment engineers present described as genuinely electric.

Version 1.0 of the MIDI specification was published in August 1983 and distributed free of charge by the MIDI Manufacturers Association — a deliberate policy choice that accelerated industry adoption. By 1984, MIDI had been implemented in the Roland TR-909 drum machine, the Yamaha DX7 synthesizer, and the Linn Electronics LinnDrum, three instruments that would define the sound of commercial pop and electronic music for the next decade. The DX7 in particular owed much of its commercial dominance to MIDI: producers could store and recall patches via SysEx over MIDI, making patch libraries a commercial product category and enabling the DX7's electric piano and marimba patches to appear on thousands of recordings. Stevie Wonder was an early adopter, incorporating MIDI-synced sequencers into live performance rigs as early as 1984.

The 1980s and 1990s saw MIDI expand from a hardware studio tool into the backbone of personal computer music. Opcode Systems' Vision (1988), Mark of the Unicorn's Performer, and later Steinberg's Cubase (1989) introduced graphical piano-roll MIDI editing to desktop computers, replacing hardware step sequencers for composition work. General MIDI (GM), ratified in 1991, standardised a 128-instrument patch map and the channel 10 drum assignment, enabling MIDI files to play back consistently across different synthesizers — the foundation for the MIDI file format used in video game soundtracks, ringtones, and karaoke systems throughout the 1990s and 2000s. The Roland Sound Canvas SC-55, released in 1991, became the reference GM device and set the tonal expectations for an entire generation of computer music.

MIDI's limitations were debated for decades before formal action was taken. The 7-bit resolution ceiling on velocity and controllers, the 16-channel cap, and the lack of bidirectional negotiation frustrated instrument designers working on expressive controllers. Interim solutions appeared: MIDI Polyphonic Expression (MPE), formalised by the MIDI Association in 2018, extended the channel model to allow per-note pitch bend, pressure, and slide on adjacent channels — enabling the Roli Seaboard, Roger Linn's LinnStrument, and Haken Audio's Continuum to express continuous multi-dimensional control per note. MPE was supported in Ableton Live 10.0.2 (2018) and Logic Pro 10.4 (2018). The full MIDI 2.0 specification, developed by the MIDI Association with contributions from Apple, Google, Microsoft, Roland, and Yamaha, was ratified in January 2020. USB MIDI 2.0 device class support arrived in Windows 11 (2023) and macOS Ventura (2022), and Logic Pro became the first major DAW to expose MIDI 2.0 resolution in its note editor in 2023.

06 How Producers Use It

For keyboard and synth programming, MIDI is the primary compositional medium. A producer records a MIDI performance into a piano roll, then iterates on it non-destructively: adjusting note lengths, correcting pitches, reshaping velocity curves, and drawing pitch bend or CC automation — all without re-recording. The performance and the sound are decoupled, so changing the synth patch or swapping the plugin entirely leaves the composition intact. Experienced keyboard programmers layer CC 1 (mod wheel) automation and pitch bend into a take in real time, capturing the expressive arc of a performance rather than a static sequence. Velocity layers in modern samplers — triggered at different velocity thresholds — mean that a well-programmed MIDI part can move through multiple articulations (piano pp to ff) within a single instrument.

Drum programming via MIDI is the dominant workflow in electronic, hip-hop, R&B, and pop production. Producers input kick, snare, hat, and percussion hits on a step grid or by finger-drumming on a pad controller, then apply velocity humanisation, timing nudges (moving individual hits ±5–15 ms ahead or behind the grid), and groove quantise templates to make the pattern feel organic. The MPC workflow — popularised by Roger Linn's Akai MPC60 and MPC3000 — uses MIDI to trigger individual samples mapped to pads, a paradigm that persists in Ableton Drum Rack, Native Instruments Maschine, and the MPC software line. Ghost notes at velocity 15–30 give snare patterns sublety; hi-hat velocity variation between triplet positions defines groove in swing-based patterns.

Live performance rigs depend on MIDI for routing, scene control, and real-time modulation. A standard live setup might assign MIDI CC 7 to a hardware fader to control instrument volume, use MIDI Program Change (PC) messages to recall synth patches at song transitions, and synchronise a hardware drum machine to the DAW via MIDI Clock. Ableton Live's Session View popularised clip-triggering via MIDI note messages from foot controllers (APC40, Launchpad), turning MIDI into a live performance matrix that can restructure a set in real time. MIDI-controlled pedal boards (Strymon Timeline, Boss ES-5) receive MIDI PC and CC messages to switch patches and set parameter values, integrating guitarists and bassists into the MIDI ecosystem.

Audio-to-MIDI conversion is a powerful compositional tool. DAWs can analyse a monophonic audio recording — a sung melody, a bass riff, a flute line — and convert the pitch and timing information into MIDI notes, which are then routed to any instrument. Ableton Live's Extract Groove function captures the rhythmic feel of a drum loop and converts it into a groove template that can be applied to any MIDI pattern, transferring the human timing nuance of a live drummer to a programmed sequence. Melodyne's DNA (Direct Note Access) technology extends this to polyphonic audio, separating individual notes within a chord voicing into editable MIDI-like blobs — a workflow that has become standard in production for correcting, transposing, and reimagining recorded performances.

AbletonUse the MIDI Effect Rack with Chord, Arp, and Velocity MIDI effects chained before a soft synth to build generative melodic patterns without touching the piano roll. The Convert Melody to MIDI function in Arrangement View is the fastest audio-to-MIDI workflow in any major DAW.
FL StudioThe Piano Roll's stamp tool and chord tools accelerate chord voicing and arpeggio construction. Use the MIDI CC automation clip (right-click any knob → Create Automation Clip) for per-parameter CC lanes that stay locked to the pattern rather than the global timeline.
Logic ProLogic's Step Sequencer (added in 10.6) provides a pattern-based MIDI drum workflow independent of the piano roll. Use Smart Controls to map CC numbers to macro knobs, then record CC automation in real time from a hardware controller for expressive plugin automation.
Pro ToolsPro Tools MIDI editor works directly in the Edit window alongside audio. Use Real-Time Properties (quantise, transpose, velocity) applied non-destructively per clip, and MIDI Beat Clock output via Setup > MIDI > MIDI Beat Clock to sync external hardware without routing audio.
ReaperReaper's MIDI routing is the most flexible in any DAW: use the MIDI routing matrix to send a single MIDI track to multiple instrument plugins on different channels simultaneously. The ReaControlMIDI and js MIDI plugins allow inline MIDI processing (transposition, CC remapping) without external utilities.

07 In the Wild

Abstract knowledge becomes practical when you can hear it in music you know. These tracks demonstrate midi used intentionally, at specific moments, for specific purposes.

Daft Punk — "Da Funk" (1995)
0:00–1:20 · Produced by Daft Punk (Thomas Bangalter & Guy-Manuel de Homem-Christo)

The central bass hook is a Roland TB-303 bassline sequenced via the 303's internal step sequencer and synced to the track via MIDI clock, with the accent and slide parameters programmed into the sequencer itself. The pitch bend between notes is a characteristic TB-303 MIDI slide — a legato portamento triggered when consecutive notes overlap in gate time. The rhythmic grid is quantised to 16th notes, but the accent velocity pattern (accents at 127, non-accents at ~80) gives the line its loping, syncopated feel. Study the note lengths: each accented note is approximately 80% of a 16th note, while non-accented notes are closer to 50%, a gating technique fundamental to acid bass programming.

Timbaland feat. Jay-Z & Missy Elliott — "Lobster & Scrimp" (2018)
0:00–0:45 · Produced by Timbaland

Timbaland's drum programming is a masterclass in MIDI velocity variation. The kick drum hits vary between velocity 100 and 120, while the snare ghost notes sit at 20–35 and the primary snare hits at 95–115 — a 4:1 velocity ratio that creates a live drum feel. Hat patterns use three distinct velocity tiers (open hats at 80–90, closed hats at 50–65, ghost hats at 15–25) to build a polyrhythmic texture from a single MIDI track. The timing of the ghost snare notes is nudged 8–12 ms behind the grid, a technique Timbaland has cited as central to his pocket. Recreate this by recording hats with a pad controller at varying pressure rather than drawing them in a piano roll.

Aphex Twin — "Xtal" (1992)
Full track · Produced by Richard D. James

An early example of MIDI-programmed ambience in which pitch bend and aftertouch data on a Roland Alpha Juno-2 create slow, continuous timbral evolution. The melody notes use channel aftertouch mapped to filter cutoff, so held notes gradually open the filter as pressure is applied — a modulation that sounds organic but is entirely MIDI-driven. The sequencer runs at a slow tempo (approximately 68 BPM) with notes at irregular rhythmic positions, avoiding the metronomic grid quality of standard 16th-note quantisation. James is known to have used Cubase on an Atari ST for MIDI sequencing during this era, exploiting the Atari's famously tight MIDI clock timing.

Kanye West — "Runaway" (2010)
0:00–0:15 · Produced by Kanye West & No I.D.

The opening piano figure — eight notes spelling out a simple melodic cell — is performed as a MIDI sequence with deliberate velocity uniformity: all notes sit at approximately velocity 65–70, creating a flat, almost mechanical affect that contrasts with the emotional weight of the track. The note lengths are slightly longer than the grid division, causing subtle overlaps on the sustain pedal (CC 64 = 127 throughout) that blur into a haze. The deliberate absence of velocity humanisation is a production choice, not a mistake — the robotic quality reinforces the track's themes of emotional numbness. When recreating this, disable velocity randomisation entirely and leave quantise at 100%.

08 Types & Variants

MIDI 1.0 (DIN / USB Class 1)
Roland JP-6 · Sequential Circuits Prophet-600 · Akai MPC60 · Roland TB-303

The original 1983 specification, transmitted at 31,250 baud over 5-pin DIN connectors or encapsulated in USB MIDI Class 1 packets. Limited to 7-bit resolution (128 steps) for all data, 16 channels per port, and unidirectional data flow. Still the universal compatibility baseline in 2025 — every synthesizer, DAW, and controller supports MIDI 1.0, making it the safest choice for cross-device interoperability.

MIDI 2.0 (UMP / USB Class 2)
Roland A-88MKII (MIDI 2.0 firmware) · Native Instruments Komplete Kontrol S-Series (MIDI 2.0 ready)

The 2020 ratified extension of the standard, using Universal MIDI Packets (UMP) over USB MIDI 2.0 Class devices. Expands velocity and CC resolution to 16-bit and 32-bit fields, introduces bidirectional device capability negotiation via MIDI-CI, and supports per-note controllers enabling continuous pitch, pressure, and timbre per note without the channel-per-note workaround of MPE. Full DAW support is rolling out through 2024–2026.

MPE (MIDI Polyphonic Expression)
ROLI Seaboard RISE · Roger Linn LinnStrument · Haken Audio Continuum · Expressive E Osmose

An extension to MIDI 1.0 that assigns each note to its own MIDI channel, enabling independent pitch bend, channel pressure, and CC 74 (timbre/slide) per note within a polyphonic performance. Supported in Ableton Live 10+, Logic Pro 10.4+, Bitwig Studio, and most modern soft synths. MPE effectively obsoletes polyphonic aftertouch hardware in most workflows, though MIDI 2.0 per-note controllers will eventually supersede MPE.

MIDI Clock & Sync
Roland TR-8S · Elektron Digitakt · Korg Volca series · Arturia DrumBrute

The timing and transport subset of MIDI System Real-Time messages, used to synchronise tempo between a DAW master and hardware drum machines, sequencers, or modular systems. The Elektron workflow — running a Digitakt or Octatrack as a MIDI clock master driving an external DAW — is common in hardware-centric live rigs. Jitter at the DAW MIDI output is the primary limiting factor; dedicated USB MIDI interfaces with hardware FIFO buffering (Focusrite Clarett, MOTU Express) provide tighter clock than onboard audio interfaces.

General MIDI (GM / GM2)
Roland Sound Canvas SC-55 · Yamaha MU90 · Korg 05R/W

A 1991 standardisation layer on top of MIDI 1.0 that defines 128 instrument patches in a fixed programme map, a standard drum map on channel 10, and minimum polyphony of 24 voices. GM was the foundation for video game soundtracks (SNES, PlayStation), karaoke systems, and early internet MIDI files. GM2 (1999) extended the patch count and added Roland GS and Yamaha XG extensions. In modern production, GM is rarely targeted except for educational software and legacy multimedia contexts.

09 Common Mistakes

  • ✕
    MIDI clock jitter from USB hubs degrading groove
    Routing MIDI clock through a cheap unpowered USB hub introduces jitter of 5–20 ms, which at 120 BPM causes drum machine hits to wander audibly within a 16th-note grid. The fix is to use a dedicated USB MIDI interface connected directly to a host USB port (not a hub), or to run the drum machine as clock master and slave the DAW to it rather than the reverse. On Windows, enabling WASAPI exclusive mode for the audio driver reduces MIDI scheduling jitter significantly.
  • ✕
    Using velocity 127 on every note, eliminating dynamics
    Beginner producers often default to maximum velocity on all programmed notes, resulting in performances that sound rigid and fatiguing. Real instruments and real players vary velocity continuously — even a metronomic drum machine programmer creates 5–15 velocity steps between hits. Apply a velocity randomiser (±10 in Ableton's Velocity MIDI effect, or the humanise function in Logic's Piano Roll) as a starting point, then manually sculpt accents above 100 and ghost notes below 40 for groove that sounds intentional.
  • ✕
    Stuck MIDI notes from missing Note Off messages
    If a DAW transport is stopped mid-note, or a cable is unplugged while a note is held, the receiving instrument never receives the Note Off message and the note continues sustaining indefinitely. The fix is to send an All Notes Off (CC 123, value 0) or All Sound Off (CC 120, value 0) message to the instrument — most DAWs do this automatically on stop, but hardware instruments with buggy MIDI implementations sometimes ignore these messages. Keeping a MIDI panic button (a toolbar macro that transmits All Notes Off on all 16 channels simultaneously) is standard practice in live performance rigs.
  • ✕
    Ignoring note length when programming melodic parts
    The default note length in most piano rolls is exactly one grid division, which produces a staccato feel even when the intent is legato. Melodic lines — basslines, lead synth hooks, string lines — benefit from notes that are 95–105% of the grid division so they overlap slightly, particularly when the instrument has a legato mode that connects adjacent overlapping notes with portamento. Conversely, plucked or percussive sounds (piano, muted guitar, staccato brass) benefit from notes at 30–50% of the grid to prevent muddiness from held releases.
  • ✕
    Overcrowding a single MIDI channel with all instruments
    Sending drums, bass, leads, and pads to the same MIDI channel causes controller data (mod wheel, pitch bend) intended for one instrument to affect all others. This is particularly destructive with pitch bend, which will detune every instrument on that channel simultaneously. Assign each instrument to its own MIDI channel and MIDI track in the DAW, using channels 1–9 and 11–16 for melodic content and reserving channel 10 for drums. In multi-timbral hardware synths, verify each part's receive channel is set uniquely.
  • ✕
    Sending CC data without understanding the target mapping
    Drawing CC 7 automation expecting to control filter cutoff will instead control channel volume on most instruments, because CC 7 is universally assigned to volume in the MIDI spec. Similarly, CC 10 controls pan, not panning on a specific layer. Before automating a plugin via MIDI CC, check the plugin's MIDI learn page to confirm what each CC number actually controls for that instrument. Misrouted CCs create confusing behaviour — volume suddenly drops mid-song, panning jumps unexpectedly — that is difficult to diagnose if the CC mapping was assumed rather than verified.

10 Producers Also Look Up

LFOLow-frequency oscillators are commonly assigned to MIDI CC destinations for modulation, vibrato, and filter animation in synth programming. Envelope (ADSR)MIDI velocity is the primary input to envelope depth scaling in samplers and synths, making velocity-to-envelope mapping a core sound design parameter. LatencyMIDI-to-audio latency — the delay between a key press and audible sound — is determined by audio buffer size and driver efficiency, and must be minimised for live performance. Sample RateWhile MIDI carries no audio, the audio sample rate of the host DAW determines the resolution of sample-accurate MIDI scheduling within each buffer cycle. OscillatorThe oscillator of a synthesizer receives MIDI note numbers as pitch CV equivalents, with each semitone step corresponding to a precise frequency ratio. FilterFilter cutoff frequency is one of the most common MIDI CC automation targets, with CC 74 assigned as the standard filter brightness controller in the MIDI spec.

12 Frequently Asked Questions

MIDI carries only performance data — note numbers, velocities, controller values, and timing information. It does not transmit any audio signal. The sound is generated by whichever instrument or plugin receives the MIDI messages and converts them into audio. This is why a MIDI cable or USB MIDI connection produces no sound on its own, and why the same MIDI performance can be redirected to a completely different instrument without altering the underlying notes.
Middle C is MIDI note number 60, but whether it is displayed as C3 or C4 depends on the DAW's octave numbering convention. Ableton Live, FL Studio, and Reason display note 60 as C5. Logic Pro, Pro Tools, and Cubase display it as C3. Roland hardware historically used C4. This discrepancy causes confusion when sharing MIDI files between applications — always verify the note number (60) rather than the octave label when communicating with other producers or hardware manufacturers.
Pitch Bend is a dedicated MIDI message type with 14-bit resolution (16,384 steps) designed specifically for smooth pitch deviation, while Continuous Controllers (CC) are general-purpose parameter messages with 7-bit resolution (128 steps) that can be mapped to any instrument parameter. Because pitch bend has its own message type, it does not consume a CC number slot and its resolution is higher — making it the correct tool for expressive pitch slides, vibrato, and whammy effects. CC is better for filter sweeps, volume fades, and any parameter where 128 steps of resolution is sufficient.
A single MIDI 1.0 port supports 16 independent channels, each addressable to a separate instrument or synth part. In practice, a multi-timbral hardware synthesizer can receive up to 16 different parts on one port, while a DAW can route a single MIDI port to 16 separate software instruments simultaneously. For more than 16 independent voices, additional MIDI ports are required — modern USB MIDI interfaces support multiple virtual ports, each with their own 16 channels.
Quantisation snaps note start times to the nearest grid division — typically 16th or 8th notes — correcting timing imperfections in a recorded performance. At 100% strength, full quantisation produces a metronomic, robotic feel that lacks human groove. Most DAWs offer a quantise strength parameter (50–90%) that moves notes partway toward the grid while preserving some of the original timing feel. Groove quantise templates, derived from the timing of sampled drum loops, apply a swing or shuffle offset to the grid itself, maintaining consistent groove while correcting egregious timing errors.
Program Change (PC) is a MIDI message that instructs a receiving instrument to switch to a specific patch number (0–127). Producers use PC messages to automate patch changes on hardware synthesizers mid-song — for example, switching a Roland JV-1080 from a string ensemble sound in the verse to a brass patch in the chorus. In live rigs, PC messages are sent from a MIDI foot controller to pedalboards and rackmount effects processors to recall saved preset configurations at song transitions. In a DAW, PC events can be inserted directly into a MIDI track's event list.
MIDI Polyphonic Expression (MPE) assigns each note in a polyphonic performance to its own MIDI channel, allowing independent pitch bend, pressure (Channel Aftertouch), and timbre control (CC 74) per note. In standard MIDI, pitch bend and aftertouch are channel-wide — bending the pitch wheel bends all notes on that channel equally. MPE enables instruments like the ROLI Seaboard to express continuous per-finger glide, pressure, and lateral slide in a way that standard MIDI cannot. MPE is supported in Ableton Live 10+, Logic Pro 10.4+, Bitwig Studio, and most modern polyphonic soft synths via their MPE mode settings.
MIDI 2.0 replaces the 7-bit (128-step) data resolution of MIDI 1.0 with 16-bit velocity (65,536 steps) and 32-bit controller values (over 4 billion steps), effectively eliminating the stepping artefacts audible in slow pitch bend and CC automation. It introduces the Universal MIDI Packet (UMP) format, allowing multiple MIDI streams to coexist in a single connection. Most significantly, MIDI 2.0 adds MIDI Capability Inquiry (MIDI-CI), a bidirectional handshake allowing devices to discover each other's features and negotiate supported profiles — something MIDI 1.0 could never do. MIDI 2.0 is fully backward compatible with MIDI 1.0 devices via a defined translation layer, so the upgrade path does not break existing gear.
