# 47. mpw_bible_writer.py v5.0 — CURRENT SPEC

## Why v5.0

v4.0 generated excellent content but had a broken HTML template — 25 missing SEO items, nav not sticky, mobile broken, tracks mismatched, related terms 404ing. v5.0 is a complete rewrite of build_html() fixing all issues.

## Word Count Target

- Floor: 5,500 words
- Target: 6,000 words
- Ceiling: 6,500 words

## Two-Pass Architecture

**Pass 1 (20,000 tokens) — Structured Data Pass**

Returns JSON only. New in v5.0:
- `track_examples` field added — title, artist, track, year, timestamp (NO spotify_uri)
- `spotify_embeds` field REMOVED
- All other fields same as v4.0

**Pass 2 (16,000 tokens) — Prose Pass**

Receives Pass 1 JSON as context (8,000 chars — increased from 4,000).
Receives locked track list extracted from Pass 1 track_examples as separate numbered list.
Cannot invent new tracks — must use exactly the tracks from Pass 1.

## Track Examples — Architecture

The AI cannot reliably produce verified Spotify track IDs — it hallucinates structurally valid but nonexistent IDs every time. Solution:

1. Pass 1 generates track_examples with title, artist, track name, year — NO URI
2. Pass 2 receives the exact same tracks as a locked numbered list — cannot swap or invent
3. build_spotify_links() generates YouTube search URLs: `youtube.com/results?search_query=Artist+Track`
4. Section heading: "Listen on YouTube"
5. Icon: YouTube red play button
6. Result: always works, no login, no 404, correct track every time
