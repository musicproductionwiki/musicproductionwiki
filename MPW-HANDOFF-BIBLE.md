## Nav Fix — Confirmed Working on Real iPhone (Session 43)

**Root cause confirmed:** Nav pills physically scrollable but active highlight stopped advancing because active pill scrolled off-screen with no auto-scroll to keep it visible. The fix is `scrollIntoView` after each active state update.

**Confirmed working JS (replace enav_js in build_js()):**
```javascript
(function(){
  var navLinks = document.querySelectorAll('.entry-nav-inner a');
  var sections = Array.from(document.querySelectorAll('.entry-section[id]'));
  var lastActive = null;
  function getActiveId() {
    var offset = 60;
    var best = null;
    for (var i = 0; i < sections.length; i++) {
      var rect = sections[i].getBoundingClientRect();
      if (rect.top <= offset && rect.bottom > offset) { best = sections[i].id; break; }
    }
    if (!best) {
      for (var j = sections.length - 1; j >= 0; j--) {
        if (sections[j].getBoundingClientRect().top <= offset) { best = sections[j].id; break; }
      }
    }
    return best || (sections[0] && sections[0].id);
  }
  function update() {
    var id = getActiveId();
    if (!id || id === lastActive) return;
    lastActive = id;
    navLinks.forEach(function(a){
      a.classList.toggle('active', a.getAttribute('href') === '#' + id);
    });
    var activeLink = document.querySelector('.entry-nav-inner a.active');
    if (activeLink) activeLink.scrollIntoView({behavior:'smooth',block:'nearest',inline:'center'});
  }
  window.addEventListener('scroll', update, {passive:true});
  window.addEventListener('touchmove', update, {passive:true});
  update();
})();
```

## Root Cause Summary — Three Bugs That Killed Every Tool

### Root Cause 1 — "" Leak in mpw_tools_v3.py

Every tool function body (triple-quoted heredoc) had a stray `""` between the `</script>` tag and the closing `"""`:
```python
    ...calc();\n</script>""
    """        ^^^ THIS
    return _wrap(...)
```
This emitted literal `""` into the HTML after every script tag → JS SyntaxError → all tool JS dead.

Found via: `grep '</script>""' mpw_tools_v3.py` — 11 matches.
Fixed by: `fix_v3_permanent.py` — pattern `</script>""\n    """\n    return _` → `</script>\n    """\n    return _`.

### Root Cause 2 — Duplicate Tools Section in Writer

Writer had `{tools_html}` in TWO places:
1. `html.replace('TOOLS_PLACEHOLDER', tools_html)` — correct, line 2364
2. `{tools_html}` in final f-string assembly — DUPLICATE, line 2511

Result: two `<section id="tools">` blocks, two `lfoCalc` definitions, duplicate DOM IDs.
Fixed by: `fix_writer_permanent.py` — removed line 2511 occurrence.

### Root Cause 3 — Single-Quoted LTIPS JS Strings

LTIPS values in `build_lfo_sync()` used single-quoted JS strings. English contractions (`you've`, `don't` etc.) terminated the JS string early → `Unexpected identifier 've'` SyntaxError.

Found via: the `you've` apostrophe was NOT in the Python source (search found 0 contractions) — confirmed it was generated by the API at runtime, embedded in the single-quoted JS string.
Fixed by: converting all 7 LTIPS values to double-quoted JS strings.
