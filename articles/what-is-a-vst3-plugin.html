What Is A Vst3 Plugin

MP
The Music Production Wiki Team
Updated May 2026 · 9 min read · Verified May 7, 2026
Quick Answer — Updated May 2026

VST3 (Virtual Studio Technology 3) is Steinberg's current standard plugin format for virtual instruments and audio effects. It improves on VST2 with lower CPU usage when idle, multiple audio buses for native sidechain support, sample-accurate automation, and resizable interfaces. If your DAW supports both VST2 and VST3 versions of a plugin, always choose VST3.

Updated May 2026. VST3 stands for Virtual Studio Technology 3 — the third major version of the VST plugin standard developed by Steinberg, the company behind Cubase. First released in 2008, VST3 defines how virtual instruments and audio effects communicate with a Digital Audio Workstation (DAW): how audio passes through the plugin, how MIDI and control signals are routed, how automation is recorded and played back, and how the plugin's user interface is displayed inside your DAW.

A Brief History of VST

The VST format originated in 1996 with VST1, released alongside Cubase. VST2 followed in 1999 and became the dominant plugin format for over a decade — virtually every plugin and every DAW (except Pro Tools and Logic Pro) ran on VST2. When Steinberg released VST3 in 2008, adoption was slow; developers had to maintain two codebases simultaneously and DAWs were cautious about migrating. By the mid-2010s VST3 was broadly supported, and by the early 2020s it became the primary format for new plugin development. Steinberg officially discontinued VST2 licensing in 2018, meaning new developers cannot release VST2 plugins without a legacy license — a clear signal that the industry is moving to VST3 exclusively.

VST3 vs VST2: What Actually Changed

Context-Aware Processing

The biggest practical improvement in VST3 is context-aware processing. In VST2, every plugin instance in your session runs continuously, consuming CPU even when no audio passes through it. With 50 plugin instances but only 10 actively receiving audio, all 50 still burn CPU. In VST3, plugins signal to the DAW when they have nothing to process, and the DAW suspends them — they use zero CPU until audio arrives. For large sessions with many quiet tracks, this can be a significant real-world improvement. This pairs well with the strategies covered in our guide on how to build a plugin chain.

Multiple Audio and MIDI Buses

VST2 was built around a simple model: audio in, audio out. Sidechain inputs required DAW-specific workarounds. VST3 natively supports multiple input and output audio buses, making sidechain routing consistent and straightforward across all compatible DAWs — no more host-specific hacks.

Sample-Accurate Automation

VST2 automation was processed at the buffer boundary — parameter changes applied at the start of each audio buffer (typically 128–2048 samples). VST3 supports sample-accurate automation, meaning parameter changes can target any individual sample within a buffer. Fast filter sweeps, sidechain ducks, and rapid volume automation all benefit from smoother, more precise playback.

Other VST3 Improvements

  • Resizable interfaces — VST3 UIs can scale, which matters on high-DPI displays.
  • Improved preset management — standardised preset handling across hosts.
  • Better channel configuration — supports mono, stereo, surround, and Atmos configurations natively.
  • MIDI 2.0 support — available in newer VST3 implementations.
Bottom line: If a plugin offers both VST2 and VST3 versions, always install and use the VST3 version. The only reason to keep a VST2 is if you have an existing project that uses it and you need session continuity.

Understanding which formats your DAW supports is essential before purchasing plugins. Logic Pro uses AU (Audio Units) exclusively on macOS — VST3 plugins will not load in Logic. AU, AAX, and CLAP: The Other Plugin Formats

AU (Audio Units) is Apple's native plugin format for macOS. It is used exclusively by Logic Pro and GarageBand. On macOS, AU is fully equivalent to VST3 in capabilities for most use cases — the choice is determined by your DAW, not by quality differences. Producers working in Logic should look for AU versions of plugins, though most major developers ship both.

AAX (Avid Audio eXtension) is Pro Tools' exclusive format. VST3 plugins do not load in Pro Tools — you need the AAX version. Most major developers (FabFilter, iZotope, Waves) release AAX alongside VST3, but smaller independent developers often skip AAX due to Avid's higher certification costs and requirements. If you use Pro Tools, verify AAX availability before purchasing. Our best plugins for mixing guide notes AAX availability for each recommendation.

CLAP (CLever Audio Plugin) is an open-source format developed as a modern alternative to both VST3 and AU. It offers similar technical capabilities — efficient processing, multiple buses, sample-accurate automation — while being completely free to implement. CLAP is gaining adoption in Bitwig Studio, Reaper, and FL Studio, and developers like FabFilter and u-he now ship CLAP versions alongside VST3.

VST3 File Locations and Installation

VST3 plugins use the .vst3 file extension. Default install locations:

  • Windows: C:\Program Files\Common Files\VST3
  • macOS: /Library/Audio/Plug-Ins/VST3

Unlike VST2, which was a single .dll (Windows) or .vst bundle (macOS), a VST3 plugin is a folder bundle that may contain the plugin binary, preset files, and documentation. Do not manually move individual files out of the bundle. Most modern installers handle placement automatically. Understanding plugin formats is foundational to getting the most from tools like the FabFilter Pro-Q 3 or iZotope Ozone 12, both of which ship in VST3, AU, AAX, and CLAP formats.

Which Format Should You Use?

Use VST3 if you are on Windows or using Ableton Live, Cubase, Studio One, Bitwig, Reaper, or FL Studio. Use AU if you are on Logic Pro or GarageBand. Use AAX if you are on Pro Tools. If CLAP is available and your DAW supports it, it is a solid choice — especially in Bitwig. Avoid installing VST2 for new plugins unless you specifically need it for an older project. Frequently Asked Questions

FAQ What is a VST3 plugin?
VST3 (Virtual Studio Technology 3) is the third major version of the VST plugin format developed by Steinberg. It is a software interface standard that lets virtual instruments and audio effects communicate with DAWs, improving on VST2 with lower CPU usage when idle, better routing, sample-accurate automation, and resizable interfaces.
FAQ What is the difference between VST2 and VST3?
VST3 adds context-aware processing (zero CPU when silent), native multiple audio buses for sidechain routing, sample-accurate automation, resizable UIs, better channel configuration, and improved preset management. VST2 runs continuously even when idle and lacks these capabilities.
FAQ Do all DAWs support VST3?
Most modern DAWs support VST3, including Ableton Live 11+, Cubase, Studio One, Bitwig, FL Studio 21+, and Reaper. Logic Pro uses AU on macOS instead, and Pro Tools uses AAX — neither supports VST3.
FAQ Is VST3 better than AU (Audio Units)?
VST3 and AU are technically equivalent for most production tasks, but serve different platforms. VST3 is cross-platform (Windows and macOS), while AU is macOS-only and used by Logic Pro and GarageBand. The choice is determined by your DAW, not by inherent quality differences.
FAQ Can I use VST2 plugins in a VST3-compatible DAW?
Many VST3-compatible DAWs also support VST2 simultaneously, including Ableton Live 11+ and Studio One. However, Cubase 13 dropped VST2 support on Apple Silicon Macs, and the industry is gradually phasing out VST2, so VST2-only plugins will eventually face compatibility issues.
FAQ What is the VST3 file extension and where are plugins installed?
VST3 plugins use the .vst3 extension. On Windows they install to C:\Program Files\Common Files\VST3 by default; on macOS to /Library/Audio/Plug-Ins/VST3. A VST3 is a folder bundle that may contain the plugin binary, presets, and documentation — do not manually split the bundle.
FAQ What is CLAP and how does it compare to VST3?
CLAP (CLever Audio Plugin) is a fully open-source plugin format offering similar capabilities to VST3 — efficient processing, multiple buses, sample-accurate automation — while being free to implement. It is gaining support in Bitwig, Reaper, and FL Studio, and developers like FabFilter and u-he now ship CLAP versions.
FAQ What is AAX and how does it differ from VST3?
AAX (Avid Audio eXtension) is Pro Tools' exclusive plugin format. VST3 plugins cannot load in Pro Tools — you need the AAX version of a plugin. Most major developers release AAX alongside VST3, but smaller developers may not due to Avid's higher certification costs.
